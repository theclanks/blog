Translation management and generation of static content with Transifex + Sphinx

 Sex 03 maio 2013 by The Clanks

Hi, my name is Luis Carlos Otte Junior, I’m Brazilian and live at São Carlos, São Paulo.

The proposed project is at two parts, considering time and difficulty level:

  1. Website static generation with translations – 4 weeks – Easy
  1. Re-write the HTML pages for the Sphinx format (.rst file).
  2. Use sphinx-build to generate the .pot files and translate them in Transifex.
  3. After generating .mo translated file use the sphinx-build to generate the pages in each language. .. [1]
  4. Use python-fabric to implement the deploy process (copying, compiling translation files, and build static files). [2]
  5. Create a html template for sphinx with Language auto-selection.
  1. Provide translated versions of images and screenshots – 8 weeks – Moderate
  1. Create an add-on for Transifex, reuse your classes (Project, user permission, and language) and implement the upload/export of the images according to the language.
  2. Create an extension in the Sphinx, to manage the correctly images at the generate static pages for each language. [3]

Why do you want to work on this project?

I would like to work in this project because a wish learn more skills, working with international project, learn more English, relate with people from other countries, and, mainly, contribute to the open source community.

Why should we choose you?

I like to work and study in all that is necessary, specially Linux, Python, and PHP. Nowadays, I’m working with open source projects in a corporation. And I would like to be useful in anything.

Please list relevant academic, industry and/or open source experience

I’m a graduate student of analysis and systems development and a developer on Virgos Co. (www.virgos.com.br) and work with Python/Django, C, and PHP development. Currently, I’m working in a data mining project with Django and Selenium RC, and others telecom projects using Asterisk, C programming and Python.

Git no ssh do Openshift
Seg 10 setembro 2012 by The Clanks

Para utilizarmos o Git direto no servidor Openshift, precisamos trabalhar com as variáveis:

  • GIT_DIR “Diretório do git”
  • GIT_WORK_TREE “Diretório raiz do projeto”

Você pode exporta-las para economizar comandos:

$ export GIT_WORK_TREE=$HOME"/blog/repo/"
$ export GIT_DIR=$HOME"/git/blog.git/"
